## Anti-Patterns Verdict

**LLM assessment:** Clears the "AI made this" squint test — glance cards are genuinely composed, data is interpreted, triage maps a real operator's Sunday. Fails the product-register bar in the seams: mismatched component vocabulary (bespoke inline-styled CorrectModal buttons vs `.btn`; two navs; three select styles), uppercase tracked micro-eyebrows as the entire labeling grammar, hero-metric cards on variance detail, an identical four-card campus grid on /variance, and glassmorphism as the default theme on every surface (committed and consistent, but never earned by the product's purpose).

**Deterministic scan:** 4 findings, 2 rules, 0 false positives — full agreement with the LLM review on the stripes, plus one catch the LLM review missed:
- `side-tab` ×3 (absolute ban): GlanceView.tsx:548 (3px urgency stripe on recommendation rows), TriageView.tsx:204 (slot header status stripe), TriageView.tsx:586 (row status stripe layered on an existing separator + fill)
- `layout-transition` ×1 (detector-only catch): WorkbenchView.tsx:368 animates `width` on the progress bar; use `transform: scaleX()`

**Visual overlays:** not available this run (see provenance note above).

## Overall Impression

The core idea is executed better than most internal tools: Glance leads with a verdict sentence, recommendations are ranked and routed, and triage embeds real domain intelligence (computed PLAN clocks, bundle-overlap hints). But the app currently ships two parallel design systems (viewer vs instrument), leaks operator plumbing onto leadership surfaces, and has one outright trust-breaker (the contradictory May-31 state). The single biggest opportunity: make the leadership path (login → verdict) flawless and jargon-free — that's the audience PRODUCT.md names.

## What's Working

1. **Verdict-first glance architecture** (GlanceView.tsx:341–353, 691–735): status pill → big number + delta → interpreting sentence → ranked, owner-routed recommendations. "Interpretation over inventory," actually executed.
2. **Triage in service-flow order with computed evidence** (TriageView.tsx:71–189): cumulative PLAN clock beside recorded ACTUAL, and overlap hints that tell the operator which button to press. Domain intelligence in the UI.
3. **Chart craft above library defaults** (ChartTooltip.tsx, GlanceView.tsx:70–117, WorkbenchView.tsx:71–242): DOM-measured tooltips, generous hit targets, labeled median reference lines, real `aria-label` summaries on every SVG.

## Priority Issues

1. **[P0] Unreachable work + contradictory status on non-production dates.** SLP · May 31: headline "All items are clear.", date picker "· 9", /variance/SLP shows "3 review · 9 unmapped", triage table renders a dead-end empty state. The tool hides queue items while advertising them — a direct hit on "trusting the numbers enough to act." Fix: make attention counts and the empty state agree; when a plan has items but no production slot, render them with an explicit explanation and action path. → `/impeccable harden`
2. **[P1] Login page layout broken at desktop widths.** Card spans ~1315px of a 1400px viewport. Cause: unlayered `.app-page{max-width:88rem}` (globals.css:69–75) beats Tailwind v4's layered `max-w-md` (login/page.tsx:16). First screen every user sees. Fix: constrain in the component's own class or move app-shell rules into a layer. → `/impeccable polish`
3. **[P1] The Guide banner buries the verdict on every instrument view** and its Workbench copy describes controls that don't exist (DashboardGuide.tsx:26–30; rendered above children in (instrument)/layout.tsx:32). Fix: collapsed by default / "?" popover in nav; correct stale copy. → `/impeccable onboard` + `/impeccable clarify`
4. **[P2] Micro-text contrast failures on glass.** ~2.3:1 for --ink-35/--ink-disabled at 11px; ~2.5:1 for --phase-mid text; ~3.6:1 review-on-fill; ~3.9:1 over-on-fill — all meaning-bearing micro text. (--ink-70 ≈ 5.8:1 is fine.) Fix: darker on-tint text variants; keep meaning-bearing text ≥4.5:1. → `/impeccable audit` then `/impeccable polish`
5. **[P2] Interaction primitives lack accessible semantics.** CorrectModal: no role="dialog"/focus trap (CorrectModal.tsx:239–258); Toast: no role="status" — every action outcome is silent to screen readers; expand/segment toggles missing aria-expanded/aria-pressed. Fix: native `<dialog>`, role="status", state attributes. → `/impeccable harden`

## Persona Red Flags

**Alex (power user):** Guide wall on every route; zero keyboard shortcuts; campus/slot/horizon switches are full server navigations; card expansion state resets each visit; Glance→Workbench link drops the date.

**Sam (screen reader / keyboard / low contrast):** silent toasts; modal focus never moves and background stays tabbable; phase-bar detail lives in title attributes on non-focusable spans; 2.3–3.9:1 micro text; "Map to…" selects labeled only by placeholder option.

**Pastor Dave (campus pastor, 90 seconds, phone):** the "/" fork ("Open instrument" is internal vocabulary) costs his first 10 seconds; then the Guide; verdict sentence requires a tap per card; "provisional target", "unmapped", "review pressure" mean nothing to him; four red OVER TARGET pills read as a scoreboard of failure (principle 5 risk). Responsive layout itself is decent — his problem is vocabulary and tap-depth.

## Minor Observations

- Two parallel design systems: (viewer) uses Tailwind utilities + glass-tile; instrument uses hand-rolled classes + hundreds of inline style objects. Drift is guaranteed.
- Zero-count "0 REVIEW"/"0 UNMAPPED" pills render in warning amber on every /variance/[campus] row — color on inactive state, alarm fatigue.
- Raw PCO plan titles ("#4", "#2 - PDarin") surface on leadership pages.
- Broadcast-trend SVG letterboxes: fixed 560×150 viewBox centered in a ~1300px card.
- Nav Triage badge uses --over red for a routine workload count.
- No prefers-reduced-motion handling anywhere (low impact today).
- Token hygiene: --ink-55 defined but unused; --amber-text/--amber-fill never referenced.
- SLP (#e76f51) and LV (#f4a261) campus dots are near-identical hues for color-blind users; labels mitigate.

## Questions to Consider

1. Why do two reporting products ship in one app? /variance and /instrument answer the same question with different navs, cards, and color semantics. Which one goes on the staff-meeting screen?
2. If a pastor can't act on "unmapped", why can he see it before the verdict?
3. All four campuses are red "OVER TARGET" every week. Is the target wrong, the threshold wrong — and how long before leaders stop seeing the color (and start feeling scoreboarded)?
4. Would the "boardroom-grade" promise survive a solid-surface variant better than 22px blur-on-blur survives a bright room and an old projector?
